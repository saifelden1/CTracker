Why this design is cheap to ship: No new tables. No migration script.

1. Add two columns to the existing SessionsTasks table: status TEXT DEFAULT 'todo' and dueDate TEXT NULL. Map the legacy progress field once: 0 → 'todo', 1–99 → 'in_progress', 100 → 'done'. Done.
2. Keep EntityDetailView, ProjectMetaData, and the right-hand Project Info panel exactly as they are.
3. Inside ProjectDetailView::setupProjectChrome(), replace the existing left-column scroll-area (built by the base class) with a new TasksBoardWidget. The base class's units/sessions data is reused — every SessionTaskData simply renders as a card in the column matching its status.
4. The Units stay — they just become the unit-tag chip at the bottom of each card. The "+ Unit" button still works. The "+ Task" button writes a row with status='todo' instead of progress=0.
5. The Courses code path is not touched at all — UnitExpandableWidget and SessionTaskRow keep doing what they do for courses.

Total scope: ~2 SQL ALTER TABLEs + 1 new widget (TasksBoardWidget) + ~30-line patch to ProjectDetailView. Everything else stays.
